Install the groundwork-statusline by configuring `~/.claude/settings.json`.

**Why a wrapper?** Claude Code does *not* expand `${CLAUDE_PLUGIN_ROOT}` for a
`statusLine` command defined in user settings (it is a hooks-only placeholder),
and hardcoding the versioned plugin path (e.g. `.../groundwork-statusline/1.2.5/...`)
makes the statusline silently keep running the *old* script after every plugin
update. So this install writes a tiny stable wrapper that always execs the newest
installed version, and points settings at that wrapper.

Steps:

1. Resolve the plugin's versions directory (the parent of the versioned install
   dir) and the absolute home dir:
   ```bash
   VERSIONS_DIR="$(dirname "$CLAUDE_PLUGIN_ROOT")"
   HOME_DIR="$HOME"
   echo "$VERSIONS_DIR"; echo "$HOME_DIR"
   ```

2. Write the wrapper to `$HOME/.claude/groundwork-statusline.sh` (substitute
   `<VERSIONS_DIR>` with the value from step 1), then make it executable:
   ```bash
   #!/usr/bin/env bash
   # Auto-generated by groundwork-statusline:install.
   # Always runs the newest installed version, so plugin updates are picked up
   # automatically without re-pointing settings.json.
   base="<VERSIONS_DIR>"
   latest=$(ls -1 "$base" 2>/dev/null | grep -E '^[0-9]+\.[0-9]+\.[0-9]+$' | sort -V | tail -1)
   script="$base/$latest/statusline-command.sh"
   [ -n "$latest" ] && [ -f "$script" ] && exec bash "$script"
   ```
   ```bash
   chmod +x "$HOME/.claude/groundwork-statusline.sh"
   ```

3. Read `~/.claude/settings.json`. Check if a `statusLine` key already exists:
   - If it exists and its `command` contains `groundwork-statusline`, the wrapper
     you just wrote (steps 1–2) is what it runs — tell the user it's already
     installed (and now up to date) and stop.
   - If it exists but points to something else, use AskUserQuestion to ask:
     "A statusLine is already configured pointing to a different command. Replace
     it with groundwork-statusline?" — if they say no, stop.

4. Edit `~/.claude/settings.json` to add (or replace) the `statusLine` key with
   exactly this value, substituting `<HOME_DIR>` with the absolute home dir from
   step 1 (use an absolute path, not `~`, to avoid expansion surprises):

   ```json
   "statusLine": {
     "type": "command",
     "command": "bash <HOME_DIR>/.claude/groundwork-statusline.sh",
     "height": 3
   }
   ```

5. Tell the user: "Statusline installed. Restart Claude Code (`/exit` then relaunch) to see it."
